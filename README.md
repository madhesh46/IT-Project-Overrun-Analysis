IT Project Cost and Effort Overrun Analysis

->Project Overview
This project analyzes 81 real-world IT software projects 
from the PROMISE Software Engineering research dataset. 
The goal is to identify the key factors that cause cost 
and effort overruns in IT projects and provide actionable 
business recommendations to reduce project risk.

This was developed as part of an Business Analytics 
program to demonstrate real-world data analysis skills 
using industry-standard tools.

->Business Problem
IT companies across India and globally face significant 
financial losses due to inaccurate project estimation 
and poor resource planning. Projects frequently exceed 
their planned budgets and timelines. This analysis 
addresses that challenge by using historical project 
data to predict which projects are at risk before 
overruns occur.

->Dataset
- Source: PROMISE Software Engineering Repository
- Size: 81 IT software projects
- Key variables: Team Experience, Manager Experience, 
  Project Effort, Duration, Programming Language, 
  Function Points

->What Was Done
The raw dataset was first cleaned and prepared in 
Microsoft Excel. Three new calculated columns were 
created to classify each project by effort size, 
experience level, and overrun risk. Pivot tables and 
charts were built to identify patterns across different 
project categories.

A four-page interactive dashboard was then built in 
Power BI to visualize the risk distribution, effort 
patterns, and experience impact across all 81 projects.

Finally, a Business Requirements Document was prepared 
summarizing the problem, findings, and recommendations 
in a format suitable for senior management and HR 
stakeholders.

->Key Findings
- 35.8 percent of all projects are at High Risk of 
  cost and effort overrun
- Projects managed by low-experience teams consistently 
  require more effort than estimated
- Manager experience is one of the strongest predictors 
  of project success
- High effort projects above 150 person-months with 
  junior teams show the highest overrun rates

->Tools Used
- Microsoft Excel for data cleaning and pivot analysis
- Power BI Desktop for interactive dashboard
- Microsoft Word for business documentation

->Project Deliverables
- Excel file with cleaned data, pivot tables and charts
- Power BI dashboard with four report pages
- Business Requirements Document (BRD)

->Business Recommendations
Based on the analysis, the following actions are 
recommended for IT project managers:

1. Assign managers with three or more years of experience 
   to all projects with effort above 100 person-months
2. Implement a risk scoring system at the project 
   planning stage to flag high-risk projects early
3. Invest in team training programs to improve 
   experience levels across the organization
4. Conduct mandatory weekly reviews for all projects 
   classified as High Risk
5. Adopt Function Points as the standard estimation 
   method across all IT projects
